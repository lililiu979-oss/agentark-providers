# Public Proof Pack Template

Use this when your work is real, but your public proof still feels weak or scattered.

The goal is simple:

- help a buyer trust you faster
- help a marketplace reviewer assess you faster
- help a cold prospect understand what is actually working

## 1. One-Sentence Workflow Claim

State one workflow you own.

Example:
`We run an AI-supported support triage workflow for SaaS teams with human escalation on sensitive cases.`

## 2. Buyer Problem

Explain the pain in plain business language.

- what is slow
- what is repetitive
- what is breaking
- what outcome the buyer wants

## 3. Workflow Boundary

Show where the agent starts and stops.

- what the agent does
- what a human reviews
- what is out of scope

## 4. Inputs

- systems accessed
- documents or data used
- permissions required

## 5. Outputs

- reports
- drafts
- completed actions
- QA notes
- structured handoff packages

## 6. Proof Assets

Include as many of these as you can:

- short demo video
- screenshots
- public docs
- sample outputs
- workflow diagram
- benchmark or case study
- QA or escalation example

## 7. Risk And Escalation

State what the system should never do alone.

- approval thresholds
- low-confidence cases
- exception classes
- failure or retry logic

## 8. Best-Fit Buyer

- team type
- company stage
- recurring workflow type
- what usually triggers the need

## 9. First Commercial Offer

Make it easy to buy.

- pilot scope
- expected timeline
- review cadence
- what success looks like

## 10. Public Links Pack

Before applying or doing outbound, make sure you have one clean set of links:

- main website or profile
- strongest repo or docs link
- strongest workflow proof link
- one sample output or demo link

If a buyer can understand your workflow, trust your proof, and imagine a first paid engagement in under five minutes, your proof pack is strong enough.
