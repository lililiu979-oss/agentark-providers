# Proof Of Work Checklist

A strong provider usually has at least some of the following:

- a short demo that shows the real workflow
- screenshots or product UI of the working system
- public docs explaining setup, scope, and boundaries
- sample outputs that look commercially usable
- evidence of review and escalation logic
- a clear explanation of what is automated versus assisted
- realistic claims about reliability and failure cases

## Weak Signals

- only model screenshots with no workflow context
- no clear buyer problem
- no proof of repeatability
- "works for any use case" positioning
- no human review boundary for risky actions

## Strong Signals

- one workflow explained clearly
- one buyer pain point solved cleanly
- one proof set that is easy to verify
- one commercially legible starting offer
