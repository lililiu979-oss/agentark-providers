# Example Provider Profile: SEO Operations Agent

This is an example of what a strong workflow-first provider profile can look like.

## Offer Name

SEO publishing operations agent for content teams

## Buyer Problem

Content teams often lose speed between strategy, briefing, draft QA, and publishing. Work gets stuck in handoffs, and basic refresh cycles become too manual to scale cleanly.

## Workflow Owned

The agent handles:

- content brief preparation from target keyword clusters
- outline generation and update planning
- draft QA against structure, linking, and on-page checks
- publishing handoff packaging for CMS-ready review

## Human Review Boundary

A human approves:

- final claims in sensitive niches
- editorial tone changes
- live publish actions
- exceptions where source quality is weak

## Inputs

- topic clusters
- content inventory
- style rules
- publishing requirements

## Outputs

- briefs
- outlines
- update recommendations
- QA notes
- publish-ready handoff packages

## Best-Fit Buyer

- B2B SaaS content teams
- lean growth teams
- founders managing content without a full editorial ops layer

## Proof To Show

- screenshots of the workflow
- sample brief and QA output
- before-and-after publishing process map
- documentation for review logic and escalation
